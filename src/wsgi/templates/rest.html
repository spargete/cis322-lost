petunia.cs.uoregon.edu:9090/rest/lost_key -- Returns the LOST public key for encryption and decryption purposes.
Request: The request does not provide values for any of the arguments though the data and signature fields must be present in the request. Response: result -- ‘OK’ or ‘FAIL’
key -- The LOST public key
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"result": "OK",
			"key": "bksaoudu......aoelchsauh"
		}
petunia.cs.uoregon.edu:9090/rest/activate_user -- Reactivates LOST access for a user or generates a new user account as needed.
Request: username -- OSNAP username to activate
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"username": "carter"
		}
Response: result -- ‘OK’, ‘NEW’, or ‘FAIL’
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"result": "OK"
		}
petunia.cs.uoregon.edu:9090/rest/suspend_user -- Revokes access for a user. If the user does not exist or is already inactive, this call has no effect.
Request: username OSNAP username to revoke
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"username": "jackson"
		}
Response: result ‘OK’
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"result": "OK"
		}
petunia.cs.uoregon.edu:9090/rest/list_products -- Requests a listing of all products in LOST based on a filter criteria.
Request: vendor -- Case insensitive string to match against vendor name
description -- Case insensitive string to match against product description
compartments -- Security tags for the asset as a json array, all tags must be matched for compartment and level.
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"vendor": "",
			"description": "notepad",
			"compartments": []
		}
Response: listing -- a json list containing the product information entries Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"listing": [
				{
					"vendor": "Dunder Mifflin",
					"description": "LOST legal size notepad",
					"compartments": []
				},
				{
					"vendor": "big n large",
					"description": "LOST legal size notepad"
					"compartments": []
				}
			]
		}
petunia.cs.uoregon.edu:9090/rest/add_product -- Adds products to LOST. Takes a JSON as input, returns either "OK" or "FAIL".
Request: new products -- a json list describing the new products
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"new_products": [
				{
					"vendor": "Dunder Mifflin",
					"description": "LOST letter size notepad",
					"alt_description": "Children’s storybook"
					"compartments": ["adm:s"]
				},
				{
					"vendor": "Stark Industries",
					"description": "Micronized Arc Reactor",
					"alt_description": "Baseball"
					"compartments": ["nrg:ts","wpn:s"]
				}
		}
Response: result -- ‘OK’, or ‘FAIL’
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"result": "OK"
		}
petunia.cs.uoregon.edu:9090/rest/add_asset -- Adds a new asset to LOST.
Request: vendor -- Vendor of the product
description -- Description of the product
compartments -- Additional compartments the asset should be classified under
facility -- Facility code the asset is initially housed at
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"vendor": "Dunder Mifflin",
			"description": "LOST letter size notepad",
			"compartments": ["wpn:ts"],
			"facility": "HQ"
		}
Response: result -- ‘OK’, or ‘FAIL’
Example:
		{
			"timestamp": "2017-02-02 06:15:13",
			"result": "OK"
		}
